How it works

Four moves from commit to live

  1. 01

    Clone

    Kettle shallow-clones your repo at its branch and records the real commit SHA.

  2. 02

    Build

    It detects the stack and builds a Docker image — or uses your committed Dockerfile as-is.

  3. 03

    Run & check

    The container starts on a reserved port with your env injected, then is health-checked.

  4. 04

    Promote

    Once healthy, the routing table flips to the new container and the old one is retired.

Architecture

One Bun process, three jobs

Control plane, edge proxy, and deploy engine — backed by SQLite and the local Docker daemon.

                 ┌──────────── kettle (one bun process) ────────────┐
 browser ──▶ :4000 control plane (API + dashboard, SQLite)         │
                 │     │                                            │
 *.krillin.local▶:8080 edge proxy ─┐ syncs routing table from DB   │
                 │                  ▼                               │
                 │  docker run ─▶ app container :20000 (from Git)   │
                 └──────────────────────────────────────────────────┘
Control plane · :4000

JSON API and React dashboard. All state in a local SQLite file.

Edge proxy · :8080/:80

Routes requests by Host header from an in-memory table.

Deploy engine

Drives git and docker to build, run, and health-check.

Get started

Running in three commands

You need Bun ≥ 1.3, Docker, git, and the Atlas repo checked out next to this one.

bun install
cp .env.example .env          # set SECRET
EDGE_ENABLED=1 bun start      # or: bun dev  (hot reload)

# dashboard → http://localhost:4000
# first visit creates the admin account
bun cli.ts login --url http://localhost:4000 \
  --email you@example.com --password '••••••'

bun cli.ts deploy my-app      # streams logs, exits when live
bun cli.ts logs my-app
bun cli.ts status
SECRET=$(openssl rand -hex 32)
EDGE_ENABLED=1
EDGE_HTTP_PORT=80
APP_DOMAIN=krillin.local
PORT_RANGE_START=20000
PORT_RANGE_END=20999

Production setup, systemd, and LAN-wide .local resolution are covered in the installation guide.
